XClipper, X (Twitter) के लिए एक high-fidelity, source-available web clipper है: एक Chrome extension जो tweets, threads और long-form articles को clean Markdown, PDF, HTML, JSON, TXT, CSV या Obsidian notes के रूप में सेव करता है। एक बार में एक post export करें, या अपने bookmarks, likes, किसी profile या home timeline से batch में export करें। सब कुछ आपके browser में चलता है: कोई X API key नहीं, कोई account नहीं, कोई server नहीं, कोई tracking नहीं, और noncommercial use के लिए free।

XClipper क्यों:

अधिकतर "tweet to markdown" tools किसी post को generic HTML-to-Markdown converter से गुजारकर वहीं रुक जाते हैं। XClipper अलग तरह से बनाया गया है:

- ऐसा output जिसे आपको साफ नहीं करना पड़ेगा। कुछ भी render होने से पहले page को typed Content AST में parse किया जाता है, इसलिए structure बचा रहता है: nested threads, quote tweets, polls, link cards और पूरे long-form Articles (headings, lists, code blocks)। t.co links real URLs में resolve होते हैं, emoji और @mentions intact रहते हैं, truncated posts expand होते हैं, और engagement bars, follow prompts तथा trackers हटाए जाते हैं।
- असली PDFs, screenshots नहीं। PDF export Chrome के native print engine का उपयोग करता है: selectable text, clickable links, embedded images और full Unicode, कोई flattened image नहीं।
- Batch, एक-एक करके नहीं। अपने पूरे Bookmarks, Likes, किसी Profile, Timeline या hand-picked Selection को progress bar, pause / resume / stop और dedup ledger वाले background job में export करें, साथ में optional Fast mode जो आपकी logged-in X session से items सीधे खींचता है और लगभग 10x speed-up देता है।
- 100% local, zero setup। कोई API keys नहीं, कोई accounts नहीं, कुछ भी आपके browser से बाहर नहीं जाता। Install करें और clip करें।

साथ में शामिल:

- सात export formats: Markdown, PDF, HTML, JSON, TXT, CSV या Obsidian को one-click handoff। Batch अलग-अलग files, एक combined file, या दोनों लिखता है (PDF को छोड़कर हर format)।
- Obsidian integration: optional vault और subfolder के साथ obsidian:// handoff, साथ में Obsidian-friendly frontmatter schema: [[@handle]] wikilinks, Dataview-friendly metadata, synthesized title और description, और custom tags list।
- Rich YAML frontmatter: author, handle, dates, source URL, content type और engagement stats, हर field अलग से toggleable, और custom filename templates में tokens के रूप में usable।
- Trigger करने के तीन तरीके: हर tweet पर inline button, toolbar popup और right-click menu।
- Link rot से बचने के लिए embedded images locally save करें, और single tweet या पूरा thread export करें।

इसके अलावा: clipboard में copy या download, optional inline engagement-stats row, और optional downloads subfolder।

उपयोग के लिए उपयुक्त:

- Obsidian और PKM workflows (साथ में Notion, Logseq, Hugo और अन्य Markdown tools)
- research और reference archiving
- AI prompts और RAG pipelines
- searchable second brain बनाना
- X के long-form content को offline सुरक्षित रखना

मौजूदा सीमाएँ:

- videos और GIFs playable media files के रूप में export नहीं होते
- Fast Batch X की internal API को सीधे call करता है, इसलिए बड़े runs X के rate limit तक पहुंच सकते हैं (soft-blocking); यह opt-in है, खुद pace करता है, और आप बाद में resume या re-run कर सकते हैं
- कुछ functionality x.com के page structure में बड़े बदलाव होने पर टूट सकती है
- अगर extension install या update करते समय x.com tab पहले से खुला है, तो export से पहले tab reload करें; यह intentional है, ताकि uninitialized page पर silent failures से बचा जा सके
