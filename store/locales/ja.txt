XClipper は、X（Twitter）向けの高忠実度でソースコードが利用可能なウェブクリッパーです。ツイート、スレッド、長文記事をクリーンな Markdown、PDF、HTML、JSON、TXT、CSV、または Obsidian ノートとして保存できる Chrome 拡張機能です。1 件ずつエクスポートすることも、ブックマーク、いいね、プロフィール、ホームタイムラインから一括エクスポートすることもできます。すべてブラウザ内で動作します。X API キー不要、アカウント不要、サーバー不要、トラッキングなし。非商用利用は無料です。

XClipper を選ぶ理由:

多くの「tweet to markdown」ツールは、投稿を汎用の HTML-to-Markdown コンバーターに通して終わりです。XClipper は違う設計です:

- 後から整える必要のない出力。ページはレンダリング前に型付きの Content AST として解析されるため、構造が保たれます。ネストしたスレッド、引用ツイート、投票、リンクカード、そして完全な長文記事（見出し、リスト、コードブロック）に対応します。t.co リンクは実 URL に解決され、絵文字と @メンションはそのまま残り、短縮表示された投稿は展開され、エンゲージメントバー、フォロー促進、トラッカーは除去されます。
- スクリーンショットではない本物の PDF。PDF エクスポートは Chrome のネイティブ印刷エンジンを使用します。選択可能なテキスト、クリック可能なリンク、埋め込み画像、完全な Unicode を含み、平坦化された画像ではありません。
- 1 件ずつではなく一括。ブックマーク、いいね、プロフィール、タイムライン、または手動で選んだ項目を、進捗バー、一時停止 / 再開 / 停止、重複排除ログ付きのバックグラウンドジョブでエクスポートできます。さらに任意の Fast モードでは、ログイン済みの X セッションから直接項目を取得し、およそ 10 倍高速化します。
- 100% ローカル、設定不要。API キーもアカウントも不要で、何もブラウザの外へ出ません。インストールしてクリップするだけです。

含まれる機能:

- 7 つのエクスポート形式: Markdown、PDF、HTML、JSON、TXT、CSV、または Obsidian へのワンクリック受け渡し。一括エクスポートでは、個別ファイル、結合ファイル、またはその両方を書き出せます（PDF 以外の全形式）。
- Obsidian 連携: obsidian:// による受け渡し、任意の vault とサブフォルダ指定、さらに Obsidian 向け frontmatter スキーマを提供します。[[@handle]] wikilink、Dataview 対応メタデータ、生成されたタイトルと説明、カスタムタグリストに対応します。
- 豊富な YAML frontmatter: 著者、handle、日付、ソース URL、コンテンツ種別、エンゲージメント統計を含み、各フィールドを個別に切り替え可能で、カスタムファイル名テンプレートのトークンとしても利用できます。
- 3 つの起動方法: 各ツイート上のインラインボタン、ツールバーポップアップ、右クリックメニュー。
- 埋め込み画像をローカル保存してリンク切れを防ぎ、単一ツイートまたはスレッド全体をエクスポートできます。

さらに、クリップボードへのコピーまたはダウンロード、任意のインラインエンゲージメント統計行、任意のダウンロードサブフォルダにも対応します。

こんな用途に最適:

- Obsidian や PKM のワークフロー（Notion、Logseq、Hugo、その他 Markdown ツールにも対応）
- リサーチと参考資料のアーカイブ
- AI プロンプトと RAG パイプライン
- 検索可能なセカンドブレインの構築
- X の長文コンテンツをオフライン保存

現在の制限:

- 動画と GIF は再生可能なメディアファイルとしてエクスポートされません
- Fast Batch は X の内部 API を直接呼び出すため、大量実行では X のレート制限（ソフトブロック）に達する場合があります。この機能は任意で、ペースを自動調整し、後から再開または再実行できます
- x.com がページ構造を大きく変更すると、一部機能が動作しなくなる可能性があります
- x.com のタブがすでに開いている状態で拡張機能をインストールまたは更新した場合は、エクスポート前にタブを再読み込みしてください。これは未初期化ページでのサイレントな失敗を防ぐための仕様です
