XClipper هو أداة قص ويب عالية الدقة ومتاحة المصدر لمنصة X (تويتر): إضافة Chrome تحفظ التغريدات والسلاسل والمقالات الطويلة كملفات Markdown نظيفة أو PDF جاهزة للطباعة أو ملاحظات Obsidian. يمكنك تصدير منشور واحد في كل مرة، أو التصدير دفعة واحدة من العلامات المرجعية أو الإعجابات أو ملف شخصي أو الخط الزمني الرئيسي. كل شيء يعمل داخل المتصفح: بدون مفتاح X API، بدون حساب، بدون خادم، بدون تتبع، ومجاني للاستخدام غير التجاري.

لماذا XClipper:

معظم أدوات قص المحتوى تمرر المنشور عبر محول عام من HTML إلى Markdown وتتوقف هناك. XClipper مبني بطريقة مختلفة:

- مخرجات لن تحتاج إلى تنظيفها. يتم تحليل الصفحة إلى Content AST ذي أنواع قبل أي عرض، لذلك تبقى البنية محفوظة: السلاسل المتداخلة، التغريدات المقتبسة، الاستطلاعات، بطاقات الروابط، والمقالات الطويلة الكاملة (العناوين، القوائم، كتل الكود). يتم حل روابط t.co إلى روابطها الحقيقية، وتبقى الرموز التعبيرية و@mentions كما هي، ويتم توسيع المنشورات المقتطعة، وإزالة أشرطة التفاعل ومطالبات المتابعة والمتتبعات.
- ملفات PDF حقيقية، وليست لقطات شاشة. يتم التصدير عبر محرك الطباعة الأصلي في Chrome: نص قابل للتحديد، روابط قابلة للنقر، صور مدمجة، وUnicode كامل، وليس صورة مسطحة.
- تصدير دفعي، وليس واحداً تلو الآخر. صدّر كل العلامات المرجعية أو الإعجابات أو ملفاً شخصياً أو الخط الزمني أو اختياراً يدوياً ضمن مهمة خلفية مع شريط تقدم، إيقاف مؤقت / استئناف / إيقاف، وسجل لإزالة التكرار، إضافة إلى وضع Fast اختياري يسحب العناصر مباشرة من جلسة X المسجلة لديك لتسريع يقارب 10x.
- محلي 100%، بلا إعداد. لا مفاتيح API، لا حسابات، ولا شيء يغادر متصفحك. ثبّت الإضافة وابدأ القص.

يتضمن أيضاً:

- تنسيقات إضافية عند الحاجة: احفظ صفحة HTML منسقة ومستقلة، أو المستند المنظم بصيغة JSON، أو نصاً عادياً، أو صف CSV واحداً للبيانات الوصفية. يكتب التصدير الدفعي ملفات منفصلة أو ملفاً مدمجاً أو كليهما.
- تكامل Obsidian: تسليم عبر obsidian:// مع vault ومجلد فرعي اختياريين، بالإضافة إلى مخطط frontmatter مخصص: wikilinks بصيغة [[@handle]]، وبيانات متوافقة مع Dataview، وعنوان ووصف مُولدان، وقائمة وسوم مخصصة.
- YAML frontmatter غني: المؤلف، اسم المستخدم، التواريخ، رابط المصدر، نوع المحتوى، وإحصاءات التفاعل، مع إمكانية تفعيل كل حقل على حدة واستخدامه كرمز في قوالب أسماء الملفات المخصصة.
- ثلاث طرق للتشغيل: زر مدمج على كل تغريدة، نافذة شريط الأدوات، وقائمة النقر بزر الفأرة الأيمن.
- حفظ الصور المضمنة محلياً لمنع تلف الروابط، وتصدير تغريدة واحدة أو سلسلة كاملة.

بالإضافة إلى ذلك: النسخ إلى الحافظة أو التنزيل، صف اختياري مدمج لإحصاءات التفاعل، ومجلد تنزيلات فرعي اختياري.

مناسب لـ:

- سير عمل PKM وتدوين الملاحظات القائم على Markdown
- البحث وأرشفة المراجع
- مطالبات الذكاء الاصطناعي وخطوط أنابيب RAG
- بناء عقل ثانٍ قابل للبحث
- حفظ محتوى X الطويل دون اتصال

القيود الحالية:

- لا يتم تصدير الفيديوهات وGIF كملفات وسائط قابلة للتشغيل
- يستدعي Fast Batch واجهة X الداخلية مباشرة، لذلك قد تصل التشغيلات الكبيرة إلى حد معدل X (حظر ناعم)؛ الميزة اختيارية، تضبط سرعتها ذاتياً، ويمكنك الاستئناف أو إعادة التشغيل لاحقاً
- قد تتعطل بعض الوظائف إذا غيّر x.com بنية صفحاته بشكل كبير
- إذا ثبّت الإضافة أو حدّثتها بينما تبويب x.com مفتوح بالفعل، فأعد تحميل التبويب قبل التصدير؛ هذا مقصود لتجنب أخطاء صامتة على صفحة غير مهيأة
