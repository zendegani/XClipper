XClipper 是一款适用于 X（Twitter）的高保真、源码可用网页剪藏工具：一个 Chrome 扩展，可将推文、线程和长文文章保存为干净的 Markdown、适合打印的 PDF 或 Obsidian 笔记。你可以一次导出一条帖子，也可以从书签、点赞、个人资料页或主页时间线批量导出。所有操作都在浏览器中完成：无需 X API 密钥、无需账号、无需服务器、无跟踪，并且非商业用途免费。

为什么选择 XClipper：

大多数剪藏工具只是把帖子交给通用的 HTML 到 Markdown 转换器，然后就结束了。XClipper 的构建方式不同：

- 输出无需再清理。页面会先被解析为带类型的 Content AST，再进行渲染，因此结构得以保留：嵌套线程、引用推文、投票、链接卡片，以及完整长文文章（标题、列表、代码块）。t.co 链接会解析为真实 URL，emoji 和 @提及保持不变，被截断的帖子会展开，互动栏、关注提示和跟踪器会被移除。
- 真正的 PDF，而不是截图。导出过程使用 Chrome 原生打印引擎：可选择文本、可点击链接、嵌入图片和完整 Unicode，而不是一张扁平图片。
- 批量导出，而不是逐条处理。将你的全部书签、点赞、某个个人资料页、时间线或手动选择的内容作为后台任务导出，支持进度条、暂停 / 继续 / 停止和去重记录；还可选择 Fast 模式，直接从你已登录的 X 会话抓取项目，速度约提升 10 倍。
- 100% 本地，无需配置。无需 API 密钥、无需账号，任何内容都不会离开你的浏览器。安装后即可剪藏。

还包括：

- 需要时可使用更多格式：保存为带样式的独立 HTML 页面、JSON 结构化文档、纯文本或包含一行元数据的 CSV。批量导出可写入单独文件、一个合并文件，或两者同时写入。
- Obsidian 集成：通过 obsidian:// 交给 Obsidian，可选 vault 和子文件夹，并提供定制的 frontmatter 架构：[[@handle]] 风格的 wikilink、Dataview 友好元数据、生成的标题和描述，以及自定义标签列表。
- 丰富的 YAML frontmatter：作者、handle、日期、来源 URL、内容类型和互动统计，每个字段都可单独开关，并可作为自定义文件名模板中的 token 使用。
- 三种触发方式：每条推文上的内联按钮、工具栏弹窗和右键菜单。
- 将嵌入图片保存到本地以避免链接失效，并可导出单条推文或整个线程。

另外：复制到剪贴板或下载、可选的内联互动统计行，以及可选的下载子文件夹。

适用于：

- PKM 和基于 Markdown 的笔记工作流
- 研究与参考资料归档
- AI 提示词和 RAG 流水线
- 构建可搜索的第二大脑
- 离线保存 X 上的长篇内容

当前限制：

- 视频和 GIF 不会导出为可播放的媒体文件
- Fast Batch 会直接调用 X 的内部 API，因此大批量运行可能触发 X 的速率限制（软阻塞）；它是可选功能，会自动控制节奏，并且你可以稍后继续或重新运行
- 如果 x.com 大幅修改页面结构，部分功能可能失效
- 如果在 x.com 标签页已打开时安装或更新扩展，请在导出前重新加载该标签页；这是有意设计的，用于避免未初始化页面上的静默失败
