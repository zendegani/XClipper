XClipper یک وب‌کلیپر دقیق و source-available برای X (توییتر) است: افزونه‌ای برای Chrome که توییت‌ها، رشته‌ها و مقاله‌های بلند را به‌صورت Markdown تمیز، PDF، HTML، JSON، TXT، CSV یا یادداشت‌های Obsidian ذخیره می‌کند. می‌توانید هر بار یک پست را خروجی بگیرید، یا از بوکمارک‌ها، لایک‌ها، یک پروفایل یا تایم‌لاین خانه به‌صورت batch خروجی بگیرید. همه چیز در مرورگر شما اجرا می‌شود: بدون کلید X API، بدون حساب، بدون سرور، بدون ردیابی، و برای استفاده غیرتجاری رایگان.

چرا XClipper:

بیشتر ابزارهای "tweet to markdown" یک پست را از یک مبدل عمومی HTML به Markdown عبور می‌دهند و همان‌جا متوقف می‌شوند. XClipper متفاوت ساخته شده است:

- خروجی‌ای که لازم نیست تمیزکاری کنید. صفحه پیش از هر رندر شدن به یک Content AST تایپ‌شده parse می‌شود، بنابراین ساختار حفظ می‌شود: رشته‌های تو‌در‌تو، توییت‌های نقل‌قول‌شده، نظرسنجی‌ها، کارت‌های لینک و مقاله‌های بلند کامل (تیترها، فهرست‌ها، بلوک‌های کد). لینک‌های t.co به URL واقعی تبدیل می‌شوند، ایموجی‌ها و @mentions دست‌نخورده می‌مانند، پست‌های کوتاه‌شده باز می‌شوند، و نوارهای تعامل، پیام‌های دنبال‌کردن و trackerها حذف می‌شوند.
- PDF واقعی، نه اسکرین‌شات. خروجی PDF از موتور چاپ بومی Chrome استفاده می‌کند: متن قابل انتخاب، لینک‌های قابل کلیک، تصاویر تعبیه‌شده و Unicode کامل، نه یک تصویر تخت.
- Batch، نه یکی‌یکی. همه Bookmarks، Likes، یک Profile، Timeline یا یک Selection دستی را در یک کار پس‌زمینه با نوار پیشرفت، pause / resume / stop و دفترچه dedup خروجی بگیرید، به‌همراه حالت Fast اختیاری که آیتم‌ها را مستقیماً از جلسه واردشده X شما می‌گیرد و حدود 10 برابر سریع‌تر است.
- 100% محلی، بدون راه‌اندازی. بدون کلید API، بدون حساب، هیچ چیز از مرورگر شما خارج نمی‌شود. نصب کنید و clip کنید.

همچنین شامل:

- هفت فرمت خروجی: Markdown، PDF، HTML، JSON، TXT، CSV یا ارسال یک‌کلیکی به Obsidian. خروجی batch فایل‌های جداگانه، یک فایل ترکیبی، یا هر دو را می‌نویسد (همه فرمت‌ها به‌جز PDF).
- یکپارچگی با Obsidian: ارسال از طریق obsidian:// با vault و زیرپوشه اختیاری، به‌همراه schema فرانت‌متر سازگار با Obsidian: wikilinkهای [[@handle]]، متادیتای سازگار با Dataview، عنوان و توضیح ساخته‌شده، و فهرست tagهای سفارشی.
- YAML frontmatter غنی: نویسنده، handle، تاریخ‌ها، URL منبع، نوع محتوا و آمار تعامل، هر کدام قابل فعال/غیرفعال‌سازی جداگانه و قابل استفاده به‌عنوان token در قالب‌های سفارشی نام فایل.
- سه روش اجرا: دکمه inline روی هر توییت، popup نوار ابزار و منوی کلیک راست.
- ذخیره محلی تصاویر تعبیه‌شده برای جلوگیری از خرابی لینک‌ها، و خروجی گرفتن از یک توییت یا کل رشته.

به‌علاوه: کپی به کلیپ‌بورد یا دانلود، ردیف inline اختیاری برای آمار تعامل، و زیرپوشه دانلود اختیاری.

مناسب برای:

- جریان‌های کاری Obsidian و PKM (همچنین Notion، Logseq، Hugo و دیگر ابزارهای Markdown)
- پژوهش و آرشیو منابع
- پرامپت‌های هوش مصنوعی و RAG pipelines
- ساخت یک «مغز دوم» قابل جستجو
- نگه‌داری آفلاین محتوای بلند X

محدودیت‌های فعلی:

- ویدیوها و GIFها به‌صورت فایل رسانه‌ای قابل پخش خروجی نمی‌شوند
- Fast Batch مستقیماً API داخلی X را فراخوانی می‌کند، بنابراین اجراهای بزرگ ممکن است به rate limit X برسند (soft-blocking)؛ این قابلیت اختیاری است، سرعت خود را تنظیم می‌کند، و می‌توانید بعداً ادامه دهید یا دوباره اجرا کنید
- برخی قابلیت‌ها ممکن است با تغییر قابل توجه ساختار صفحه x.com از کار بیفتند
- اگر هنگام نصب یا به‌روزرسانی افزونه یک تب x.com از قبل باز است، قبل از خروجی گرفتن تب را reload کنید؛ این کار عمدی است تا از خطاهای خاموش در صفحه مقداردهی‌نشده جلوگیری شود
